# su-PathComponentArray v0.1 Development Digest

## 概要
`su-PathComponentArray` は、SketchUp上で選択したパスに沿って ComponentInstance
を複製配列するための SketchUp Ruby Extension である。v0.1 では MVP として、
「選択した ComponentInstance 1つを、選択した単一 Edge 上に等ピッチで複製配置
する」機能を実装した。

## 開発目的
SketchUp 2025 向けの Ruby Extension として、任意パスに沿ってコンポーネントを
複製配列するプラグインを開発する。初回リリース（v0.1）では機能を絞り、単一
Edge に沿った等ピッチ配列という最小構成の MVP を確立することを目的とした。

## v0.1で実装したこと
- SketchUp Ruby Extension構成（ローダー `.rb` + 実装フォルダ `su_path_component_array/`）
- `SketchupExtension` による拡張登録（name: `su-PathComponentArray`, version: `0.1.0`, author: `airesearchagl-art`）
- メニューからの実行（`Extensions > su-PathComponentArray > Create Path Component Array`）
- `UI.inputbox` による設定入力（pitch / start_offset / end_offset / follow_path / angle_offset_degrees / group_result）
- ComponentInstance 1つ + Edge 1本の選択チェックと警告メッセージ
- 単一 Edge 上の等ピッチ配置（`PathSampler` による配置点計算）
- `follow_path` による Edge 方向追従（XY 平面上の Z 軸回転を想定）
- `angle_offset_degrees` による追加回転
- `group_result` 対応（生成結果のグループ化）
- `start_operation` / `commit_operation` による Undo 対応（エラー時は `abort_operation`）
- README整備（インストール手順・使い方・MVP制限・今後の拡張予定）
- ローカル開発用 `setup_local_dev.ps1` / `update_local_dev.ps1`
- ダブルクリック更新用 `update_local_dev.bat`

## 採用した重要判断
- v0.1 は単一 Edge 対応に絞る（複数Edge / Polyline / Curveは対象外）
- 初期UIは HTML Dialog ではなく `UI.inputbox`
- 配置基準は ComponentInstance の原点固定（中心合わせ・端部合わせは対象外）
- 角度追従は主に XY 平面上の Z 軸回転を想定（任意3D方向の完全姿勢制御は対象外）
- Claude Code クラウド版と Windows ローカル SketchUp 環境は別環境として扱う
- ローカル clone + symlink + update script（`.ps1` / `.bat`）運用を採用
- ローカル clone 先は OneDrive 対象の Documents を避け、`.claude\projects`
  配下の固定パスに変更
- merge後の通常運用は `main` を追従する（作業ブランチではなく main を pull）

## v0.2への引き継ぎ
v0.2 では、単一 Edge 対応を維持したまま、端点同士が接続した連続する複数
Edge / Polyline を1本のパスとして扱えるようにする。初回の v0.2 範囲は、
分岐なし・不連続なし・非ループの Polyline 対応に限定し、閉じたループ・
分岐パス・Curve・ランダムピッチ・角度段階変化・HTML Dialog 化・生成結果の
再編集は引き続き対象外とする。
